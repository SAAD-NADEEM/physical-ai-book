# Agent Context Update: Docusaurus-based Technical Book

## Technologies to Add to Agent Context

### Core Technologies
- Docusaurus v3.x - Static site generator for documentation
- React - Component-based UI library used by Docusaurus
- Node.js v18+ - Runtime environment for Docusaurus
- npm/yarn - Package managers for dependencies
- GitHub Pages - Static hosting platform

### Domain-Specific Technologies
- ROS 2 (Robot Operating System 2) - Robot application development
- Gazebo/Unity - Simulation environments for robotics
- NVIDIA Isaac - AI-powered robotics platform
- Vision-Language-Action (VLA) Systems - Multimodal robotics control

### Documentation Structure
- Module-based organization: 4 modules covering 13 weeks
- Chapter-based content: 1500-3000 words per chapter
- Exercise framework: 3 exercise types per chapter (logical, conceptual, implementation)
- Frontmatter metadata: title, description, keywords, sidebar_position on every page

### Constitutional Requirements
- Glossary Enforcement: All terms defined in glossary.md
- Mathematical Validation: All equations validated with citations
- Functional Code Examples: All examples tested and functional
- Academic Citation Standard: APA format for all external concepts
- Evidence-Based Technical Claims: All claims supported by citations or proof
- Version Specification Compliance: All dependencies with explicit versions
- Search optimization: Keywords in headings, metadata, and first paragraph
- SEO compliance: Automatic sitemap generation and proper robots.txt
- All code examples must run successfully in test environments
- All internal and external links must be validated
- Strict spell-check enforcement

## Context Update Instructions
When the update-agent-context.sh script is run, it should incorporate these technologies and requirements into the agent's context for future development tasks related to this project.
